# Muwa — кофейня-кондитерская в Самаре

> **Keep muwa and save peace.**

Сайт самарской кофейни-кондитерской **Muwa**: спешелти-кофе, домашние десерты и сытная
кухня. Две точки — на **Чапаевской, 92** и **Некрасовской, 2**. Сайт показывает витрину
дня, принимает предзаказ «с собой», продаёт подарочные сертификаты и собирает анкеты
в команду.

Стек: **Next.js 16 (App Router) + TypeScript + Tailwind CSS**. Адаптивная вёрстка,
фирменная палитра, шрифты Oswald / Golos Text / Nunito с поддержкой кириллицы,
компонентный дизайн и аккуратная работа с мобильным Safari (safe-area, восстановление
скролла, отключение авто-tel-ссылок).

## ✨ Что есть на сайте

- **Витрина дня** — три вкладки (десерты / кухня / напитки) с фото, описанием, КБЖУ и
  ценой; позиция добавляется в предзаказ одним тапом с анимацией «полёта в корзину».
- **Заказ с собой** — пошаговый предзаказ: корзина → выбор филиала и времени → контакты →
  оплата → номер заказа и QR-код для получения на точке.
- **Подарочные сертификаты** — выбор номинала и пожелания, на выходе — промокод.
- **Кофе недели** — отдельный блок с обжаркой и способом заваривания.
- **Правила дома** — ценности и атмосфера заведения.
- **«Хочу в команду»** — анкета для соискателей (имя, возраст, соцсети, телефон).
- **Филиалы и контакты** — переключение между точками, адреса, часы работы, телефоны,
  карта с переходом в Яндекс.Карты.
- **Наличие в реальном времени** — позиции, попавшие в стоп-лист, автоматически
  помечаются как недоступные (интеграция с YTimes, см. ниже).
- **SEO** — мета-теги, Open Graph, разметка Schema.org (`CafeOrCoffeeShop`),
  `sitemap.xml` и `robots.txt`, canonical-адреса.

## 🚀 Запуск

Нужен **Node.js 20+**.

```bash
npm install       # установить зависимости (один раз)
npm run dev       # режим разработки → http://localhost:3000
```

Продакшн-сборка:

```bash
npm run build
npm start
```

### Через Docker

```bash
docker compose up --build     # → http://localhost:3000
docker compose down           # остановить
```

## ⚙️ Конфигурация (`.env`)

Все переменные **опциональны** — сайт запускается и работает без них. Внешние сервисы
подключаются по мере готовности ключей, код витрины при этом не меняется. Полный список
с комментариями — в [.env.example](.env.example).

- **`NEXT_PUBLIC_SITE_URL`** — публичный адрес сайта (для canonical / Open Graph / sitemap).
- **YTimes** (`YTIMES_API_KEY`, `YTIMES_SHOP_CHAP`, `YTIMES_SHOP_NEKR`, …) — источник
  наличия/стоп-листа по каждому филиалу. Без ключа наличие берётся из локального
  `data/mock-stoplist.json`, а dev-эндпоинт `POST /api/dev/stoplist` позволяет менять его
  для проверки витрины. Как только задан боевой ключ и GUID точек — клиент сам
  переключается на реальный API.

## 🖼️ Фотографии

Изображения лежат в `public/assets/`. Чтобы заменить или добавить фото — положите файл
туда же с тем же именем (см. `public/assets/README.md`). Отсутствующие фото автоматически
заменяются плейсхолдером «фото скоро».

## 📁 Структура

```
src/
├── app/                 App Router: layout, page, actions, api/, sitemap, robots
│   └── api/             availability (наличие) + dev/stoplist (правка стоп-листа)
├── components/
│   └── MuwaSite.tsx     основной клиентский UI (витрина, предзаказ, сертификаты, анкета)
└── lib/
    ├── menu.ts          данные витрины и филиалов
    ├── availability.ts  расчёт наличия по стоп-листу
    └── ytimes/          клиент YTimes (реальный + заглушка) и конфиг
public/assets/           фотографии, логотип, иконки
```

## 🧾 Реквизиты и оплата

Оплата картами и через **СБП / T-Pay**, все цены — в рублях. Юридические документы
(оферта, политика конфиденциальности, согласие на обработку ПДн, оплата и возврат)
доступны в подвале сайта.
